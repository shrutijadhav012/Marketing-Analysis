# Marketing Analytics  Project

## Project Overview
This end-to-end data analytics project simulates a real-world business scenario for an online retail company. The business is facing a decline in **customer engagement** and **conversion rates** despite increased marketing investment. This project demonstrates how to move from a high-level business problem to a data-driven technical solution using the modern data stack.

## Business Objective
The project aims to identify the root causes of declining performance by:
* **Analyzing the Conversion Funnel:** Pinpointing where potential customers drop off.
* **Enhancing Engagement:** Identifying which marketing channels and content types drive the highest interaction.
* **Decoding Customer Sentiment:** Using Python to process feedback and reviews into actionable insights.

## Tech Stack & Methodology
* **Problem Definition:** Understanding stakeholder requirements and defining key metrics (KPIs) like Conversion Rate, Average Order Value, and Customer Feedback Score.
* **Data Cleaning (SQL):** Extracting and preparing raw marketing data for reliable analysis.
* **Advanced Analytics (Python):** Performing sentiment analysis to enrich customer review data.
* **Visualization (Power BI):** Building an interactive dashboard to track performance and support decision-making.
* **Data Storytelling:** Translating technical findings into clear business recommendations for stakeholders.

## Data Sources
The project utilizes a relational database structure containing:
* **Fact Tables:** Customer Journey, Engagement Data, and Customer Reviews.
* **Lookup Tables:** Customer demographics, Geography, and Product catalogs.

## Project Structure
- `fact_customer_reviews`: Raw and processed datasets(.csv).
- `fact_customer, fact_engagement`: Scripts for data extraction and cleaning(.sql).
- `customer_reviews_enrichment`: Notebooks for sentiment analysis(.py).
- `Marketing_Analysis_Dashboard`: The final dashboard file (.pbix).
